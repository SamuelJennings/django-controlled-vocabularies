# Issue tracker

Issues and pull requests are tracked in **GitHub Issues** on this repository, driven with the
`gh` CLI.

- **New work** starts as an issue, triaged with the labels in [`triage-labels.md`](triage-labels.md).
- **Pull requests** target `main` and must pass the required status checks before merge. `main`
  is protected by a ruleset (see [`../../AGENTS.md`](../../AGENTS.md) → CI checks) and merges are
  gated by review.
- **One PR per change.** Keep the diff scoped to a single issue or feature.

There is no external tracker; GitHub Issues is the single source of truth for planned work.
